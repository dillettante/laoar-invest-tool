# 라오어 투자 도구 (Laoar Invest Tool)

라오어式 3가지 투자법 — **VR 적립식 · VR 거치식 · 무한매수법(v2.2)** — 을 한 화면에서 다루는 웹 도구.
종목·차수별로 계좌를 여러 개 만들고, 티커만 넣으면 시세를 자동으로 불러와 **오늘 걸 LOC 주문**과 신호를 계산합니다.

- **무의존**: Python 표준 라이브러리만 사용. 설치할 패키지 없음.
- **무상태 서버**: 서버는 시세/환율 조회와 계산만 합니다. **당신의 계좌·원장은 서버에 저장되지 않고 브라우저(localStorage)에만** 있습니다.
- **멀티 계좌 · 멀티 종목**: TQQQ, SOXL 등 아무 티커나. USD + 원화(KRW) 병기.

---

## ⚠️ 면책조항 (반드시 읽어주세요)

- 이 도구는 **정보 제공·자문 보조용**이며, **투자 자문·권유가 아닙니다.** 모든 투자 판단과 결과의 책임은 사용자 본인에게 있습니다.
- **주문·체결은 사용자가 직접** 합니다. 이 도구는 자동으로 주문을 넣지 않습니다.
- 라오어 및 그 공식 서비스(카페·팬딩 등)와 **아무 관련이 없습니다.** 전략 규칙은 커뮤니티에 공개된 **2차 자료를 교차검증**해 구현한 것으로, 공식 원본과 다를 수 있습니다.
- 특히 **무한매수법 v2.2의 일부 수치는 출처마다 상충**합니다(아래 "전략 파라미터 주의"). **실제 운용 전 반드시 라오어 카페 원문으로 확정**하세요.
- 3배 레버리지 ETF(TQQQ·SOXL 등)는 **원금 손실 위험이 매우 큽니다.**

## 실행

```bash
python3 server.py            # http://localhost:8770 (LAN 노출: 0.0.0.0)
python3 server.py 8000 127.0.0.1   # 포트 8000, 로컬 전용
```

브라우저에서 주소를 열면 됩니다. Python 3.9+ 면 충분합니다(추가 설치 없음).

---

## 사용법

1. **+ 새 계좌** → 이름·전략(VR적립/VR거치/무한매수)·티커·현재 상태 입력.
2. 저장하면 티커로 시세를 자동 조회해 신호·V밴드·LOC 사다리(무한매수는 오늘의 매수/매도 주문)를 계산.
3. **체결 반영**으로 실제 체결을 기록하면 보유·평단·POOL(무한매수는 누적매수·사이클)이 갱신됩니다.
4. 계좌를 여러 개 만들어 종목·차수별로 운용. 데이터는 이 브라우저에만 저장됩니다(기기·브라우저 바뀌면 사라짐 — 중요 기록은 따로 백업).

## 배포 (외부 공개)

서버가 순수 stdlib라 어디든 올라갑니다.

- **로컬/집 LAN**: `python3 server.py` 후 같은 네트워크에서 `http://<서버IP>:8770` 접속.
- **클라우드 호스트(Render/Fly/Railway 등)**: 저장소를 올리고 시작 명령을 `python3 server.py $PORT 0.0.0.0` 로 지정. (호스트가 `$PORT`를 주입)
- **터널(외부 임시 공개)**: 로컬에서 실행 후 `cloudflared tunnel --url http://localhost:8770` 또는 `ngrok http 8770`.
- **HTTPS**: 공개 시 호스트가 제공하는 https를 쓰세요(브라우저 혼합콘텐츠 방지). 터널 도구는 https URL을 자동 제공합니다.

### 규모가 커지면
사용자가 많아지면 Yahoo 시세 요청이 레이트리밋에 걸릴 수 있습니다. 그때는 `server.py`의 `yahoo_last`에 **티커당 수 분 캐시**를 추가하세요(소규모에선 불필요).

## 전략 파라미터 주의 (무한매수 v2.2)

기본값은 커뮤니티 **주류값**이며, UI 고급설정에서 바꿀 수 있습니다. 상충/불확실 지점:

- **별지점(큰수) 산식**: `A − B·T` = 기본 **15 − 1.5·T** [주류]. 일부 출처는 `10 − 0.5·T`. → `A/B` 조정 가능.
- **종목별 익절**: TQQQ **15%** / SOXL **20%** / 기타 10% (자동 세팅, 조정 가능).
- **쿼터매도**(매일 보유 1/4 LOC)와 **쿼터손절**(40회 소진·미익절 시 보유 1/4 MOC 손절)은 구현돼 있으나, 세부는 버전(v2.2/v3/v4)마다 달라 **카페 원문 확인 권장**.
- v3.0(20분할)·v4.0(리버스·소진모드)은 미구현. 현재 v2.2 기준.

VR은 시트 V5.0.1 로직 기준(밴드 ±15%/±25% 선택, POOL 바닥: 적립 25%·거치 50%·인출 75%).
